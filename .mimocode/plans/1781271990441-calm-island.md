## T1: Fix Shuffle Mode (Bug Fix)
**Root cause**: `PlayerProvider.toggleShuffle()` delegates to `just_audio.setShuffleModeEnabled()`, but `next()`/`previous()` manually manage `_currentIndex` sequentially, ignoring just_audio's shuffled order.

**Fix approach ŌĆö Option A (recommended)**: Read current position from just_audio's `sequenceStateStream` instead of manual index tracking.
- In `_initStreams` ŌåÆ `_sequenceStateSubscription`: update `_currentIndex` from `sequenceState.effectiveSequence.indexOf(sequenceState.currentSource)` and `_currentSong` from the playlist
- `next()`: call `_audioService?.seekToNext()` when shuffle is on (let just_audio pick the next shuffled track), sync `_currentIndex`/`_currentSong` from stream
- `previous()`: call `_audioService?.seekToPrevious()` similarly
- `_handlePlaybackCompleted()`: rely on just_audio's auto-advance when shuffle+loop is on
- Fallback: if `just_audio_background` integration changes things, adjust accordingly

**Files**: `lib/providers/player_provider.dart` (lines 508-561, 608-612, 135-154, 170-189)

## T3: Implement Favorites Feature (NEW)
**Zero infrastructure exists.** Uses Kugou API for server-side playlist management.

**API Endpoints** (from user documentation):
- `POST /playlist/add` ŌĆö Create/favorite playlist (params: `name`, `list_create_userid`, `list_create_listid`, `type`, `is_pri`)
- `POST /playlist/del` ŌĆö Delete/unfavorite playlist (params: `listid`)
- `POST /playlist/tracks/add` ŌĆö Add songs to playlist (params: `listid`, `data` format: `songName|hash|albumId|albumAudioId`)
- `POST /playlist/tracks/del` ŌĆö Remove songs from playlist (params: `listid`, `fileids`)
- `GET /user/playlist` ŌĆö Get user's playlists (already implemented)
- `POST /playlist/detail` ŌĆö Playlist detail (already implemented)
- `GET /playlist/track/all` ŌĆö Playlist songs (already implemented)

**Existing API client methods** (need parameter updates to match actual API):
- `collectSheet()` / `uncollectSheet()` / `addSheetTracks()` / `delSheetTracks()` ŌĆö currently use wrong param names (`specialid`/`audio_ids`), need to match API docs

**Implementation**:
1. **Update `kugou_api_client.dart`** ŌĆö Fix existing methods + add new ones:
   - `createPlaylist(name, {isPri, type})` ŌåÆ `POST /playlist/add`
   - `deletePlaylist(listid)` ŌåÆ `POST /playlist/del`
   - `addPlaylistTracks(listid, data)` ŌåÆ `POST /playlist/tracks/add`
   - `deletePlaylistTracks(listid, fileids)` ŌåÆ `POST /playlist/tracks/del`
   - Fix `collectSheet`/`addSheetTracks`/`delSheetTracks` param names to match API

2. Create `lib/data/repositories/favorites_repository.dart` ŌĆö Local cache of favorite song IDs (SharedPreferences), sync with API on login

3. Create `lib/providers/favorites_provider.dart` ŌĆö ChangeNotifier with `toggleFavorite(song)`, `isFavorite(songId)`, `favorites` list, `createPlaylist(name)`, `addToPlaylist(listid, song)`

4. Create `lib/modules/user/favorites_page.dart` ŌĆö Show user playlists (from `getUserPlaylist`), tap to view songs, support create new playlist, support add/remove songs

5. Wire `user_center_page.dart` favorites button to navigate to `FavoritesPage`

6. Add heart icon toggle to `song_list_item.dart` and `full_player.dart`

7. Add `/favorites` route + sidebar tap handler

## T4: Implement Download Feature (NEW)
**Zero infrastructure exists.** Downloads directly from the resolved song playback URL (not Kugou cloud API).

**Download source**: Use the `song.url` (already resolved via `KugouApiClient.getSongUrl()`) to download the MP3 file directly via `dio`.

**Implementation**:
1. Create `lib/data/models/download_task.dart` ŌĆö model with song, progress (0.0-1.0), status (waiting/downloading/completed/failed), localPath
2. Create `lib/data/repositories/downloads_repository.dart` ŌĆö SharedPreferences-based (simpler than SQLite for this use case, consistent with history_repository pattern) for tracking download state + file paths
3. Create `lib/services/download_manager.dart` ŌĆö handles actual file download using `dio` (already in pubspec), stores to `getApplicationDocumentsDirectory()/downloads/`, reports progress via Stream
4. Create `lib/providers/downloads_provider.dart` ŌĆö ChangeNotifier managing download queue + status
5. Register in MultiProvider
6. Create `lib/modules/user/downloads_page.dart` ŌĆö list of downloads with progress indicators, play locally (using `song.localPath`), delete
7. Wire `user_center_page.dart` downloads button to navigate to `DownloadsPage`
8. Add download icon to `song_list_item.dart` context menu

**Files**:
- NEW: `lib/data/models/download_task.dart`
